# 1.4 Цель, задачи и границы разработки

Целью выпускной квалификационной работы является разработка web-платформы для проведения учебных соревнований по информационной безопасности в формате Attack/Defense CTF.

Объектом разработки является процесс организации и проведения практических соревнований по информационной безопасности. Предметом разработки является программная платформа, обеспечивающая регистрацию команд, управление раундами, проверку сервисов, прием флагов и отображение результатов соревнования.

Для достижения цели были поставлены следующие задачи:

1. Проанализировать особенности формата Attack/Defense CTF и определить основные сущности предметной области.
2. Сформировать функциональные и нефункциональные требования к учебной платформе.
3. Спроектировать архитектуру системы, включающую backend, frontend, checker и демонстрационные vulnbox-сервисы.
4. Разработать модель данных для хранения команд, участников, сервисов, раундов, флагов, отправок и статусов сервисов.
5. Реализовать backend API для регистрации, аутентификации, отправки флагов, управления командами, сервисами и раундами.
6. Реализовать checker-систему, выполняющую размещение и проверку флагов в демонстрационных уязвимых сервисах.
7. Реализовать frontend-приложение для игроков и организаторов соревнования.
8. Подготовить локальный Docker Compose-контур для запуска проекта.
9. Выполнить базовое тестирование реализованных функций и описать ограничения текущей версии.

Границы разработки определяются учебным характером проекта. В текущей версии система ориентирована на локальный запуск и демонстрацию ключевого сценария Attack/Defense CTF. В качестве vulnbox-среды используются демонстрационные сервисы, общие для локального Docker-контура. Полная изоляция сервисов по командам на уровне отдельных виртуальных машин или контейнеров рассматривается как направление дальнейшего развития.

Практическая значимость работы заключается в том, что разработанная платформа может быть использована как основа для учебных тренировок, демонстрации принципов Attack/Defense CTF и дальнейшего развития инфраструктуры киберполигонов. Проект также полезен как пример комплексной разработки: он объединяет backend, frontend, базу данных, контейнеризацию, checker-логику и intentionally vulnerable services.
