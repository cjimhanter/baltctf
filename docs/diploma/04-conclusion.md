# Заключение

В ходе выпускной квалификационной работы была разработана учебная web-платформа для проведения соревнований по информационной безопасности в формате Attack/Defense CTF. Разработанная система объединяет backend, frontend, базу данных, checker-сервис и демонстрационные уязвимые сервисы, что позволяет воспроизвести основной цикл соревнования в локальном окружении.

В рамках работы были проанализированы особенности формата Attack/Defense CTF, выделены ключевые сущности предметной области и сформированы требования к платформе. На этапе проектирования была предложена архитектура, включающая Django backend, Vue frontend, PostgreSQL, checker и demo vulnbox-сервисы. Также была разработана модель данных для хранения команд, участников, настроек соревнования, заявок, сервисов, раундов, флагов, отправок и статусов сервисов.

В результате реализации backend обеспечивает регистрацию и аутентификацию пользователей, управление командами и участниками, административные сценарии, генерацию флагов, прием и проверку отправок, запуск checker tick и формирование агрегированных данных для dashboard, scoreboard и service status. Frontend предоставляет отдельные маршруты для dashboard, полной таблицы результатов, матрицы сервисов, командного кабинета и панели администратора. Checker выполняет реальные HTTP-операции против демонстрационных vulnbox-сервисов и сохраняет результаты проверок.

Практическая ценность проекта состоит в возможности использовать его как учебный стенд для демонстрации принципов Attack/Defense CTF. Платформа показывает связь между атакой, защитой, доступностью сервисов и итоговым результатом команды. Благодаря Docker Compose проект можно развернуть локально и использовать для демонстрации без сложной инфраструктурной подготовки.

Дальнейшее развитие платформы может включать перенос checker-задач в асинхронную очередь, изоляцию vulnbox-сервисов для каждой команды, расширение набора demo-сервисов, добавление полноценного e2e-тестирования, развитие scoring model и более полную локализацию backend-сообщений. Эти улучшения позволят приблизить учебный прототип к инфраструктуре, пригодной для проведения более масштабных соревнований.
