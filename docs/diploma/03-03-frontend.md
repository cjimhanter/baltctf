# 3.3 Реализация frontend-части на Vue 3

Frontend-часть реализована на Vue 3 с использованием Vite. Приложение является single-page application и взаимодействует с backend через REST API. Такой подход позволяет быстро обновлять состояние соревнования без полной перезагрузки страниц и разделить серверную бизнес-логику от клиентского интерфейса.

Основной shell приложения находится в `frontend/src/App.vue`. Он содержит верхнюю навигацию, переключатель языка и `router-view`, в котором отображаются route-level страницы. Маршрутизация реализована через `vue-router`. В текущей версии доступны маршруты `/`, `/scoreboard`, `/services`, `/team` и `/admin`.

Главная страница dashboard отображает сводку соревнования, компактную таблицу результатов, аналитику по сервисам, timeline checker-проверок и историю отправок. Отдельная страница `/scoreboard` показывает полную таблицу результатов с расширенными метриками. Страница `/services` предназначена для просмотра матрицы статусов сервисов и истории проверок по раундам.

Командный портал `/team` объединяет сценарии регистрации, входа, просмотра команды, управления участниками и отправки флагов. Если пользователь не авторизован, он видит guest access flow. После входа frontend загружает пользовательский контекст и показывает рабочую область команды. Для капитана доступны дополнительные действия по управлению составом.

Панель `/admin` предназначена для staff-пользователей. В ней организатор может просматривать состояние соревнования, работать с командами, сервисами, заявками и раундами, а также запускать checker tick. Frontend не является источником прав доступа: все административные операции дополнительно проверяются backend-ом.

Для повторного использования логики создан composable `useCompetitionPage.js`. Он выступает orchestration entrypoint и собирает состояние, загрузчики данных, derived values, mutation-функции, auth/team actions и admin actions. После разрастания логики отдельные части вынесены в модули `competitionPage*.js`: factories, state, derived, loaders, mutations, auth/team actions и admin actions.

Интерфейс поддерживает два языка: русский и английский. Словарь находится в `frontend/src/i18n.js`, выбранный язык сохраняется в `localStorage`, а frontend отправляет его в заголовке `Accept-Language`. Это позволяет постепенно развивать серверную локализацию без изменения клиентской архитектуры.

Стили организованы по SCSS 7-1 architecture. В проекте используются слои `abstracts`, `base`, `components`, `layout`, `pages`, `themes` и `vendors`. Для классов применяется BEM-подобное именование. Такой подход облегчает сопровождение интерфейса, поскольку стили компонентов, страниц и базовых элементов разделены по назначению.

Для frontend-тестирования используется Vitest, `@vue/test-utils` и `jsdom`. В проекте добавлены тесты для app shell, страниц и composable-логики. Это помогает проверять, что route-level страницы корректно монтируются, а общая логика загрузки и изменения состояния не ломается при дальнейших изменениях.
